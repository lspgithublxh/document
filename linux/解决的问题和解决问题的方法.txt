  文件都有权限，归属用户、归属群组、其他用户。三种权限，三种人，不一定都有。
  一旦命令会操作文件，都会触发该文件对三大用户的三种权限和本用户所有权限的对比。

8.执行命令的身份
  1.$在命令行输入处前面：表示是一般用户。
  2.切换到root用户：先设置root密码： sudo passwd  然后输入当前用户的密码，回车会叫输入root的密码。（这样就设置root用户权限成功了）
    再切换到root用户：su root 输入刚才输入的密码。$变成#。则切换为了root用户身份。

9.shell console terminal
  shell只是命令输入窗口：--传给系统
  console 系统自带的控制台。可以输入命令，且有系统的回显。
  terminal 终端，可以在另一台机器上的一个控制台。远端输入命令，并且有系统的回显。
    安装终端如：sudo apt-get install rxvt-unicode


10.网络服务：（解决两个问题：虚拟机上网，虚拟机与实体机通信，文件共享，实体机中浏览器访问到虚拟机中的web服务器）
   10.1配置一个新的连接（实际上这个连接会加到实体机上）。默认安装的时候，会产生两个连接（eth1,eth8）,都是局域网连接。
   10.2网络连接模式：
	a.主机是不固定外网IP模式（DHCP方式）：NAT模式（默认方式，是共享主机的ip地址）VMnet8。利用了DHCP。不用自己配置，即虚拟的DHCP服务器会分配ip,网关，子网掩码给虚拟机。（主虚之间可以互相访问）
						不能同局域网内其他真实主机通信，互联网上也不能访问到虚拟机。
	b.主机有固定的外网IP模式（）：桥接模式（无线连网方式）。（且ip设置为同一网段的另一个ip地址，网关、子网掩码、DNS地址设置一样）（类似和主机构成一个局域网，并且虚拟机就是模拟的局域网中的另一台主机）。而主机可以通过路由器访问外网，虚拟机也可以。并且可以当做虚拟服务器，为局域网内其他机器提供网络服务。
					比如不止开启一个虚拟机，而开启了多个虚拟机。VMnet0。VMnet0就是虚拟交换机。
					此模式下基本上都可以互相访问。
	c.与主机构成局域网模式（）：host-only模式。ip和主机不在同一个网段（导致虚拟机只能连接主机，不能连接外网）。但是虚拟机的ip和虚拟网关、虚拟dns,虚拟dhcp地址在同一网段。这种方式利用VMnet1这个网络（局域网）的DHCP服务器（是虚拟的）来动态分配的（用户不用自己设置配置了）。
		
	
	VMnet8 > VMnet1 > VMnet0 逐渐手工配置多。
	用来寄生 》 用来构建局域网 》用来构建能上外网的局域网
	主机防火墙打开，虚拟机也访问不到主机（或者上网）。

12.机器IP
  一台主机，可以同时有多个ip。除了一个外网ip外，在每一个局域网（虚拟局域网）都有一个ip（每个局域网的区别是：网段不一样。不在同一个网段上，就在不同的局域网上。一个网段就可以建立一个局域网）。
    同时，每一个ip都对应有网卡。外网ip对应本机的实际网卡（物理网卡），在每个虚拟局域网中的ip则对应本机的虚拟网卡。本机的虚拟网卡连上的是虚拟局域网中的虚拟路由器（虚拟网络），该路由器另外连了其他的虚拟网卡（往往是一台虚拟机的网卡--即虚拟网卡）。
   (同时还有虚拟DHCP服务器，这些都是软件模拟出来的（VMware软件模拟出来的，总共模拟出了三个虚拟网络：VMnet8,VMnet1,VMnet0.每个虚拟网络核心都是一个虚拟路由器。用户新建的虚拟机可以配置连接到这三个网络之一,同时配置该虚拟机的ip地址（即在所选择连接到的网络中的地址），实体机中也可以配置实体机在这三个网络中的ip地址（内网ip地址））。。)。。


   12.1 实体机配置：在虚拟网络中的ip.（windows网络连接窗口中可以进行）
   12.2 用户建立的虚拟机配置：在虚拟网络中的ip.（进入虚拟机用命令行工具进行配置：或者说控制台）
   12.3 VMware软件配置：三个虚拟网络的种种参数。（比如是否启用DHCP服务，指定网段（子网）（比如：192.168.59.0/24）(192.168.59.0),指定网关（在NAT设置按钮下弹出指定））
        			（端口转发配置），使得外部软件可以访问到虚拟机。

14.路由器（网关）或者说虚拟路由器（网关）
   14.1 路由器（网关）有两种端口：LAN局域网端口和WAN外网端口。路由器可以从外网端口向外网连着的机器发送数据，该机器网段是另一个网段。
   14.2 路由器有两个ip。一个是内网ip，一个是外网ip。内网ip和外网ip不在一个网段上。但是，路由器可以将局域网机器（路由器局域网端口）传来的数据转发到另一个局域网机器（从路由器局域网端口，到内网中机器的网卡），也能转发到外网（从广域网端口，到外网中机器的网卡）的直接连着的那台机器（的网卡）。反过来当然也可以。
	其中，路由器要实现将从局域网端口上传来的数据转发到广域网端口上，这个路由器需要有NAT（网络地址转换）的功能。否则就不能。
   14.3 当然，路由器还可能直接或间接提供有DNS，DHCP服务。

   具体例子：vmware中:
           NAT模式：虚拟出了一个有NAT服务的路由器。（有DHCP服务）（宿主机可以设置在该局域网的ip）（虚拟出了一个网络|宿主机可见的网络连接）。Vmnet8
	   Host-Only模式：虚拟出了一个没有NAT服务的路由器。（有DHCP服务）（宿主机可以设置在该局域网的ip）（虚拟出了一个网络|宿主机可见的网络连接）。VMnet1
           桥接模式：可以认为虚拟出了一个没有NAT服务的路由器，但是没有提供DHCP服务。（宿主机没有局域网ip可以设置）（因为没有虚拟出一个网络，所以宿主机看不到任何网络连接）.Vmnet0
		      这个路由器一端已经连着了宿主机的物理网卡，一端连着虚拟机的虚拟网卡，第三端连着外网。路由器的这三端都是一个ip，且虚拟机的网卡的ip和物理网卡的ip在同一个网段。
			所以外网中的机器可以用ip区分访问到虚拟机和宿主机。
   14.4 通过路由器访问外网并传回数据，不一定要有外网ip。但是外网中的机器要精确访问到某一台机器，那么机器必须要有外网ip，但是从定义上看就没有，所以外网中的机器不能访问到局域网中的某台机器。如果路由器两端的机器都有外网ip，那么这个路由器真的就是internet上的路由器了。！！！
