1.分区问题：
  windows先定分区，后定目录。且分区加目录就是路径。
  linux先定目录，后定分区。且路径与分区无关。分区挂在目录下。

 6.1 配置文件的配置方式：

 6.2可执行文件：
    由文件的执行操作权限决定。不是扩展名

7.权限
  文件都有权限，归属用户、归属群组、其他用户。三种权限，三种人，不一定都有。
  一旦命令会操作文件，都会触发该文件对三大用户的三种权限和本用户所有权限的对比。

8.执行命令的身份
  1.$在命令行输入处前面：表示是一般用户。
  2.切换到root用户：先设置root密码： sudo passwd  然后输入当前用户的密码，回车会叫输入root的密码。（这样就设置root用户权限成功了）
    再切换到root用户：su root 输入刚才输入的密码。$变成#。则切换为了root用户身份。

9.shell console terminal
  shell只是命令输入窗口：--传给系统
  console 系统自带的控制台。可以输入命令，且有系统的回显。
  terminal 终端，可以在另一台机器上的一个控制台。远端输入命令，并且有系统的回显。
    安装终端如：sudo apt-get install rxvt-unicode


10.网络服务：（解决两个问题：虚拟机上网，虚拟机与实体机通信，文件共享，实体机中浏览器访问到虚拟机中的web服务器）
   10.1配置一个新的连接（实际上这个连接会加到实体机上）。默认安装的时候，会产生两个连接（eth1,eth8）,都是局域网连接。
   10.2网络连接模式：
	a.主机是不固定外网IP模式（DHCP方式）：NAT模式（默认方式，是共享主机的ip地址）VMnet8。利用了DHCP。不用自己配置，即虚拟的DHCP服务器会分配ip,网关，子网掩码给虚拟机。（主虚之间可以互相访问）
						不能同局域网内其他真实主机通信，互联网上也不能访问到虚拟机。
	b.主机有固定的外网IP模式（）：桥接模式（无线连网方式）。（且ip设置为同一网段的另一个ip地址，网关、子网掩码、DNS地址设置一样）（类似和主机构成一个局域网，并且虚拟机就是模拟的局域网中的另一台主机）。而主机可以通过路由器访问外网，虚拟机也可以。并且可以当做虚拟服务器，为局域网内其他机器提供网络服务。
					比如不止开启一个虚拟机，而开启了多个虚拟机。VMnet0。VMnet0就是虚拟交换机。
					此模式下基本上都可以互相访问。
	c.与主机构成局域网模式（）：host-only模式。ip和主机不在同一个网段（导致虚拟机只能连接主机，不能连接外网）。但是虚拟机的ip和虚拟网关、虚拟dns,虚拟dhcp地址在同一网段。这种方式利用VMnet1这个网络（局域网）的DHCP服务器（是虚拟的）来动态分配的（用户不用自己设置配置了）。
		
	
	VMnet8 > VMnet1 > VMnet0 逐渐手工配置多。
	用来寄生 》 用来构建局域网 》用来构建能上外网的局域网
	主机防火墙打开，虚拟机也访问不到主机（或者上网）。
