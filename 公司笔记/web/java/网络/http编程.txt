1.后台请求，并且登录
  ---不只是页面请求，而且是一般的数据请求。
	(认为数据请求和webservice请求应该是一样的，都是一样的http协议，只是内容格式。 
	只是webservice这是一种区别与springmvc / struts的处理请求（包括客户端请求的报体格式和服务端返回的报体格式）的方式。
	springmvc:请求和返回的报体都是纯粹的内容，没有其他信息。而像请求不同的数据，是通过不同的请求路径决定的.
	strusts:同上。
	webservice:url基本是固定的，而若干个不同的请求其实共用了一个url，但是报体中区分了请求哪个具体的方法，而且自然也含有数据。返回也是，也含有数据和具体的请求方法。)
	-----当然，一些公司会自己开发出类似于springmvc或者webservice的http请求处理方式的处理程序---部署在http服务器上。
