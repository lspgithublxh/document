1.本质：是有单一操作系统的并在该操作系统上创建出了大量的相互隔离的环境（文件系统）（容器）的虚拟机（VM）。轻型虚拟机。
  每个容器都可以部署/安装大量的应用。每个容器之间相互隔离，互不知道。


2.层次：应用（web、数据库、hadoop集群、消息队列） ----- 容器 ------- 操作系统
   容器之间没有任何接口。

3.c/s架构方式（客户端，主机），客户端通过api与主机上的docker守护进程通信，如创建docker容器。
   3.1使用docker镜像创建容器。一个docker镜像创建多个docker容器。
   3.2docker镜像是创建容器的模板，docker仓库中有大量的docker镜像。
   3.2安装docker，用docker machine这个命令行工具。

4.安装和启动...：（最新的ubuntu上）
     4.1 sudo apt-get install docker.io
     4.2 sudo service docker start
     4.3 sudo docker version
      搜索相关镜像：sudo docker search centos

10.安装和启动：在root用户目录下：
  10.1 查看安装后的状态：docker status
  10.2 立即先启动（才可以使用其他命令）：docker start   （另一种运行方式：update-rc.d docker defaults）
  10.3 下载镜像：docker pull ubuntu
  10.4 查看镜像和自己的版本：docker images   ; docker version
  10.5 利用下载的镜像，运行它产生一个容器：docker run -i -t ubuntu  并且进入到了ubuntu容器中。（一个容器一个进程）
  10.6 运行镜像--即会进入该容器，如果没有会自动下载它：docker run -it centos /bin/bash
  10.7 查看还在运行中的容器进程：docker ps

  10.7 网络连接： nc -U /var/run/docker.sock 两次回车

11. 循环安装的问题：ubuntu里安装docker ,docker里安装centos容器，centos容器里再安装docker。。。。

14. 下载镜像的过程：
    客户端（用户在客户端输入脚本，输入命令）发送信息到docker deamon, deamon 发送信息到 docker hub, 拉下来 镜像后创建容器，并运行容器，产生信息后还发送给客户端。
