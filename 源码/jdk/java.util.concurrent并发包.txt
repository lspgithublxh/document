		>原子地保证成功地CAS更新为输入值：@getAndSet(int)返回以前的值。一直CAS尝试，直到成功。
			>转发@unsafe.getAndSetInt(this, valueOffset, newValue)
				>循环中：先取@this.getIntVolatile(Object, long) 后用来更：@this.compareAndSwapInt(Object, long, int:get, int:set)更新成功推出循环，否则继续取-更。显然，第一个方法是可见性的获取，所以如果其他类更新了，那么读取的时候回因为失效而重新从内存中加载。
				>返回更新成功之前的老值。
		>原子地只尝试一次CAS更新为输入值：@compareAndSet(int expect, int update)
			>直接@unsafe.compareAndSwapInt(this, valueOffset, expect, update)结果直接返回。 
		>原子地保证成功地CAS更新为+1的值：@getAndIncrement() 。保证策略：循环尝试，直到成功。
			>直接转发@unsafe.getAndAddInt(this, valueOffset, 1)
				>循环中：同理也是先可见性地取：@this.getIntVolatile(Object, long) 后 +1后更：@this.compareAndSwapInt(Object, long, int:get, int:set)成功则返回，否则则继续取-更。
		>原子地保证成功地CAS更新为-1的值：@getAndDecrement()。保证策略：循环尝试，直到成功
			>类似：直接：@unsafe.getAndAddInt(this, valueOffset, -1)
		>同上，类似要更新为增加一个增量delta:@getAndAdd(int delta)
			>类似，直接@unsafe.getAndAddInt(this, valueOffset, delta)
		>原子地保证成功地先增1后取：@incrementAndGet()
			>利用@unsafe.getAndAddInt(this, valueOffset, 1)返回的是老值，显然老值只比新值小1：所以直接老值+1即可。而就不是再去读---因为可能已经被并发地修改了。
		>原子地保证成功地先减1后取：@decrementAndGet()
			>直接@unsafe.getAndAddInt(this, valueOffset, -1) - 1即可。 
		>同上，原子地一定成功地更增一个增量：@addAndGet(int delta)
			>直接@unsafe.getAndAddInt(this, valueOffset, delta) + delta即可。
		>原子地一定成功地CAS更新为映射规则运算出的新量：@getAndUpdate(IntUnaryOperator) 入参就是映射规则：即@IntUnaryOperator.applyAsInt(int)方法返回也是一个int
			>先取：@get()
			>后调用运算规则运算得出要映射为的值：@IntUnaryOperator.applyAsInt(int)
			>将此值CAS更新回去：@compareAndSet(prev, next) 如果成功了则退出，否则继续循环。返回自然是老值。
		>同上，如果返回新值：那么@updateAndGet(IntUnaryOperator)仅仅修改返回值即可。
		>同上类似，如果是规则是二元运算-还需要注入一个参数：那么就是@getAndAccumulate(int,IntBinaryOperator)道理类似。

			>初始化一个空的对象数组：读索引和放索引都指向0，空标记为true，满标记为false
			>放元素的过程：先看满标记，如果false则放，然后看放索引+1==取索引？是则表示满了，更新满标记=true，否则先加元素后将放索引+1(原子更)。
			>取元素的过程：先看空标记，(而不是再次:放=取 比较一下看)，如果false则先取元素，然后先读索引+1，再看读索引==取索引？是则更新空标记为true。
	>业务模型的API定义：
		>带元素的队列的构造：@ArrayBlockingQueue(int,boolean,Collection)
			>用可重入锁 ReentrantLock:lock来同步  
				>迭代集合，就非常普通的将各个元素先检查非空： @checkNotNull(var7)-一个空直接抛出异常， 后放入items对象数组即可：最后更新this.count=总共增加的数量，this.putIndex=最后的元素所在的索引+1位置。
		>满了抛异常的添加：@add(E var1)
			>直接转发：@super.add(var1) 
				>直接offer:@this.offer(var1) 如果成功了返回true, 否则抛出异常：new IllegalStateException("Queue full")
		>添加元素：@offer(E var1) 显然无阻塞
			>非空检测：@checkNotNull(var1)
			>用 ReentrantLock.lock来同步 
				>非满检查：this.count != this.items.length 满了就直接返回了。
					>直接入队：@this.enqueue(var1)
						>非常普通的实现：就是直接放：this.items[this.putIndex] 后 putIndex区间内自增处理,this.count自增1。然后非空信号的触发：@this.notEmpty.signal()让等待元素的线程唤醒一个来读取元素。
		>满了先阻塞地添加元素：@put(E var1) 
			>用 ReentrantLock.lock来同步：可中断的同步：@lockInterruptibly()  
				>循环中：看是否满：this.count == this.items.length如果成立则满了，那么阻塞：this.notFull.await() 
				-此时则没有满：于是添加元素：@this.enqueue(var1)  因为在同步中，所以此时醒来的只有一个。(其他的唤醒了但不是head的继任者所以还是继续等待)
		>限时提供：@offer(E,long,TimeUnit)
			>非空检测：@checkNotNull(var1)
			>用 ReentrantLock.lock来同步 :可中断的同步：@lockInterruptibly() 
				>循环中：看是否满：this.count == this.items.length 如果true则限时等待：this.notFull.awaitNanos(var5)
				-此时则没有满：@this.enqueue(var1)入队添加元素 
		>拉取元素：@poll()
			>用 ReentrantLock.lock来同步:lock()
				>检查是否有元素：this.count == 0 如果有：则@this.dequeue()并将值返回。否则返回null
					>弹出元素过程也很简单：直接先取this.items[this.takeIndex] 后置空var1[this.takeIndex] = null 再 this.takeIndex 区间内自增1。非常奇怪的：是检查this.itrs != null 则会：this.itrs.elementDequeued()。 最后的事情：信号触发唤醒一个等待取的线程：this.notFull.signal()
		>限时拉取：@poll(long, TimeUnit)
			>用 ReentrantLock.lock来同步：可中断的同步：@lockInterruptibly()  
				>循环中：元素为0则限时等待：this.count == 0 则 this.notEmpty.awaitNanos(var4)
				-此时则非0了：开始获取元素：@this.dequeue()并返回。
			-总结：和@offer()是对应的。 由它所在线程唤醒。
		>阻塞式获取：@take()
			>用 ReentrantLock.lock来同步 :可中断的同步：@lockInterruptibly() 
				>循环中：如果this.count == 0则：阻塞等待@this.notEmpty.await()
				-此时则有数量了：且唤醒了，则直接取元素@this.dequeue()返回
		>只读元素：@peek()
			>用 ReentrantLock.lock来同步:lock()
				>直接取元素来返回：@this.itemAt(this.takeIndex)返回。
		>移除指定元素：@remove(Object)
			>用 ReentrantLock.lock来同步:lock()
				>算法也是很平常的: 从 [takeIndex,putIndex] 逐个元素来@equals()来比较，如果相等则 移除@this.removeAt(int)
					>定索引的移除算法：如果 索引==takeIndex 那么：直接将所在位置清空，后 takeIndex自增1，count自减1,类似可能@this.itrs.elementDequeued()调用。
						>否则索引不是 takeIndex: 那么移除则需要整理，这个index肯定在[takeIndex,putIndex]之间，所以做法是：将index后的到putIndex为止的元素向左移动1位，然后之前的最后一位元素设置null, putIndex更新为新的，而count也自减1，也可能@this.itrs.elementDequeued()调用
						>最后一定会做的：@this.notFull.signal()通知。
		>是否包含：@contains(Object)
			>元素为空直接返回：
			>否则：用 ReentrantLock.lock来同步:lock()
				>如果数量大于0：则开始从 [takeIndex,putIndex] 逐个元素 比较，相等则返回。
		>转换为数组：@toArray()
			>用 ReentrantLock.lock来同步:lock()
				>新建一个count大小的数组：@new Object[this.count]
				>如果this.items.length - this.takeIndex > count 那么自然要分两段来复制：否则只需要一段直接赋值： @System.arraycopy(this.items, this.takeIndex, var1, 0, var3)
		>转移到集合里：@drainTo(Collection)
			>直接转发到：@this.drainTo(var1, Integer.MAX_VALUE)
				>检查集合非空：@checkNotNull(var1)
				>用 ReentrantLock.lock来同步:lock()
					>非常直接的，从takeIndex开始，转移count个元素到集合里：即有一个循环过程。
					>转移了这么多个，那么this.count 也要减少这么多个。this.takeIndex也要更新设置为最后取走元素的位置。
					>最后是非空激发：循环中，如果条件var7 > 0 && lock.hasWaiters(this.notFull)满足：那么就需要激发一次：this.notFull.signal()，同时var7也要减少
>写时复制列表：CopyOnWriteArrayList 的实现  
	>业务模型： 
		>概念： 
			>对象数组： 可见性volatile, 不序列化 trasient 
			>同步锁：可重入锁 
			
	>业务模型的API定义：对于简单的基本的不叙述。
		>初始化构造：@CopyOnWriteArrayList(Collection)
			>入参转为数组：@.toArray()
			>复制全部元素返回一个新数组：@Arrays.copyOf(var2, var2.length, Object[].class)
			>新数组就是 列表的数组属性：@this.setArray(var2)
			-总结两种数组复制：一种:利用一个数组返回第二个数组，；第二种：输入两个数组，一个赋值给另一个。
		>复制方法的覆盖：@clone()	
			>调用父类Oject.clone()这个native方法完成基本的复制。
			>得到的对象调用@resetLock()来对对象的lock 可重入锁属性进行新设置：UNSAFE.putObjectVolatile(this, lockOffset, new ReentrantLock())
		>转换为数组：@toArray()
			>获取数组属性array:返回一个新的：@Arrays.copyOf(Object[], len)
				>先获取输入数组的一个副本：看入参==Object[].class是否是对象数组类型，是则直接new Object[], 否则@Arrays.newInstance(var2.getComponentType(), len)
				>然后直接调用系统的方法来给新数组赋值：@System.arraycopy(var0, 0, var3, 0, Math.min(var0.length, var1)) 这是一个native方法 
		>设置元素方法：@set(int var1, E var2)--肯定会新建数组。为什么要新建数组，因为@get()元素方法是没有lock同步的-避免脏读(读未提交？)。但也有问题：并行设置的时候返回的元素就不一定是旧元素。
			>使用lock:ReentrantLock属性进行同步：
				>先查看var1位置如果是var2元素：那么什么也不改变，直接返回旧元素。
				>如果不是：则复制该数组出一个新的数组，在var1位置设置注入的值，再将这个新数组的引用替换为原数组的引用即可。
		>添加新元素方法：@add(E var1)
			>同样用lock:可重入锁进行同步 
				>对本数组复制一个长度+1的新数组：Arrays.copyOf(var3, var4 + 1)， 即最后一个位置的值是null。(因为System.arraycopy时候不会设置到这里)
				>再对这个新数组设置var1:var5[var4] = var1
				>再替换本数组的引用：@this.setArray(var5)
		>指定位置添加元素方法：@add(int var1, E var2)
			>通用用lock:可重入锁进行同步。
				>算法类似：但是直接调System.arraycopy()来复制两半分别到新数组，中间设置新元素。如果var1是末尾，那么类似上述方法，直接Arrays.copyOf()即可。
		>移除指定位置元素：@remove(int var1)
			>通用用lock:可重入锁进行同步。
				>如果是移除0位置元素：也是直接先复制原数组1...end的一个新数组设置回去。
				>否则不是从0开始：则新建数组，而分两半进行复制进去，再将新数组设置回去。
		>移除某个区间：@removeRange(int var1, int var2)
			>通用用lock:可重入锁进行同步。
				>类似也是：如果区间末尾是end,那么直接@Arrays.copyOf()前面的部分为一个新是数组设置回去。否则分两半进行复制。
		>不存在则添加：@addIfAbsent(E var1, Object[] var2)类似， 也是先看是否和数组内元素equals，如果是的，那么直接返回，否则复制一个长度+1数组， 新元素加到末尾。当然过程要同步。
		>批量移除：@removeAll(Collection<?> var1) 就是一个个不包含在入参集合则转移到原长新数组，然后实际添加数量从新数组复制出到一个新数组返回@Arrays.copyOf()即可。
		>遍历满足条件则消费：@forEach(Consumer<? super E> var1)。方法很简单，就是不加锁的遍历和调用消费器@.accept(var5)。这里也说明为什么添加/删除元素等要新增数组，因为这里有消费--如果原数组直接修改而元素还没有复制完全而就开始遍历，那么就会脏读而有问题。
		>满足条件则移除：@removeIf(Predicate<? super E> var1)：类似，但是要加锁，先将不满足的转移到新数组，然后按转移个数而复制出一个新数组，设置回去。
		>逐个映射替换：@replaceAll(UnaryOperator<E> var1)：类似，加锁，新建数组，逐个@apply()出新的元素来设置到新数组，然后新数组设置回去即可。
		>排序方法：@sort(Comparator<? super E> var1)
			>使用lock:ReentrantLock进行同步：
				>先复制一份数组副本，再排序：@Arrays.sort(var5, var1)
					>如果没有比较器：使用默认的排序方法@sort(var0) --内部用二分排序。
					>否则有比较器，如果@LegacyMergeSort.userRequested为true,那么使用排序：@legacyMergeSort(var0, var1)
						>先克隆一个新数组：@var0.clone()
						>如果比较器为空：采用归并排序：@mergeSort(var2, var0, 0, var0.length, 0)
							>如果数量小于7：则用冒泡排序
							>否则数量大于7：则分为两半各进行一次递归调用@mergeSort(dest, src, low, mid, 0)
							-此时则可以认为通过分解的方式已经将两个长度<7的排序好了，那么开始归并：
							>如果此时mid位置的元素小于mid+1位置的元素：那么说明两个窗口数组已经拍好序了，直接将这两个窗口的数组复制到一个入参dest数组@System.arraycopy(src, low, dest, destLow, length)后返回即可。
							>否则进行归并：即用两个指针p,q分别指向两个窗口的起点，开始一次遍历，而将结果复制到dest数组里。
						>否则：@mergeSort(var2, var0, 0, var0.length, 0, var1)
					>否则使用：TimSort.sort(...)排序 
						>排序的长度<2:直接返回
						>排序长度<32: 使用二分排序：
							>先计算@countRunAndMakeAscending()
								>先断言：assert lo < hi 如果为false则会抛出错误：AssertionError
								>lo/hi相邻：直接返回1

								>否则值更大：也进入while循环，找到值开始减少的位置runHi
								>则直接返回runHi - ro 相当于顺序排好的长度。
							>将结果带入二分排序：@binarySort(a, lo, hi, lo + initRunLen, c)	
								>显然要断言：assert lo <= start && start <= hi
								>从start开始遍历到hi:start位置的元素是第一个未排序的，而左边的是排好序的元素，因此：
									>先用二分查找的方式先找到当前待排序的元素应该插入的位置：用while即可找到：
									>然后复制插入位置右边的元素：统统向右移动一位。这里有些小动作：如果右边只有1个2个元素，那么直接交换，否则用@System.arraycopy(a, left, a, left + 1, n)
									>最后将当前待排序元素设置到插入位置：start++开始下一个待排序的元素
						>否则是长数组：先new TimSort<>(a, c, work, workBase, workLen)
						>计算最小长度：@minRunLength(len): 即len的最高6位的一半+ 前6位之后的值里如果有1个1则都是1否则是0
						
				>最后将新数组设置回去。
